
# Error: An error is a problem in your program that stops execution or causes unexpected behavior.
Syntax Errors - These happen when your code is written incorrectly.
Runtime Errors - These happen while the program is running.
Logical Errors (Most Dangerous) - Code runs without crashing, but result is wrong.

# Debugging :- Debugging is the process of finding, analyzing, and fixing errors (bugs) in your code.
# Debugging is the systematic process of identifying, analyzing, and fixing bugs using tools like breakpoints, stack traces, logging, and runtime inspection.

Ways to Debug in Node.js
There are 4 main methods:
                          1. Console logging
                          2. Node built-in debugger
                          3. VS Code debugger (most powerful)
                          4. Chrome DevTools debugging
                          
# What Is a Breakpoint? :- A breakpoint is a marker where execution pauses.
